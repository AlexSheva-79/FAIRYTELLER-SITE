# Инструкция: как вручную добавить новую работу в галерею

Эта инструкция — для добавления работы в **полную галерею** (`gallery.html`).
Если нужно добавить работу ещё и в блок «Избранное» на главной (showcase) —
это отдельный, более сложный процесс (нужно выбрать акцентный цвет рамки и
т.д.), в конце инструкции есть короткая заметка про это.

Всего 4 шага: **подготовить изображения → положить файлы → добавить запись
в код → проверить и опубликовать**.

---

## Шаг 1. Подготовь изображение

### 1.1 Придумай slug (техническое имя файла)

Slug — это короткое имя на английском, латиницей, в нижнем регистре, слова
через дефис, с порядковым номером в начале. Посмотри последний номер в
`js/works.js` (сейчас последняя работа — `13-neon-fighter`) и возьми
следующий по порядку.

Примеры хороших slug: `14-broken-mirror`, `15-city-of-glass`.

**Важно:** slug должен быть одинаковым везде — в именах файлов и в записи
кода (шаг 3). Ошибка в написании — самая частая причина, почему картинка
не отображается.

### 1.2 Сделай два размера изображения

Тебе нужно два файла из одной картинки, оба в формате **WebP**:

| Версия | Ориентировочный размер | Где используется |
|---|---|---|
| **preview** | вписать в рамку ~800×1000 | превью-карточка в галерее |
| **full** | вписать в рамку ~1600×2000 | полноразмерный показ в лайтбоксе (по клику на карточку) |

Оба размера — это **не жёсткий кроп**, а «вписать с сохранением
пропорций» (как в PowerPoint «вписать в рамку»): если у картинки другие
пропорции (например, квадрат или сильно вытянутый портрет), она просто
впишется в указанные габариты без обрезки содержимого.

### 1.3 Конвертируй в WebP через squoosh.app

1. Открой https://squoosh.app в браузере
2. Перетащи туда исходное изображение (PNG/JPG)
3. Справа выбери формат **WebP**
4. Выставь качество **80–85**
5. Если нужно — измени размер (вкладка Resize) под нужные габариты (800×1000
   или 1600×2000, вписав с сохранением пропорций)
6. Скачай результат (кнопка со стрелкой вниз)
7. Повтори для второй версии (preview и full — это два отдельных экспорта
   с разным размером)

---

## Шаг 2. Положи файлы в нужные папки

Переименуй оба скачанных файла в `{slug}.webp` (например,
`14-broken-mirror.webp`) и положи:

- **preview**-версию → `assets/works/gallery/preview/{slug}.webp`
- **full**-версию → `assets/works/gallery/full/{slug}.webp`

Имя файла должно **точно** совпадать со slug, включая регистр букв.

---

## Шаг 3. Добавь запись в `js/works.js`

Открой файл `js/works.js`. Это массив `WORKS` — каждая работа там
описана вот таким блоком:

```js
{
  slug: "14-broken-mirror",
  title: { ru: "Разбитое зеркало", en: "Broken Mirror" },
  tags: { ru: ["портрет", "разбитое стекло"], en: ["portrait", "broken glass"] },
  showcase: false,
  dateAdded: "2026-07-05",
  w: 800, h: 1000,
},
```

Скопируй этот блок и вставь его **перед закрывающей квадратной скобкой**
массива `WORKS` (то есть в самый конец списка работ, после последней
существующей записи, перед строкой `];`). Затем заполни поля:

| Поле | Что писать |
|---|---|
| `slug` | Твой slug из шага 1.1, в кавычках |
| `title.ru` / `title.en` | Название работы на русском и английском |
| `tags.ru` / `tags.en` | Теги-ключевые слова. **Очень важно:** оба массива должны содержать **одинаковое количество тегов в одинаковом порядке** — тег `tags.en[0]` обязан быть переводом `tags.ru[0]`, и так далее. Теги используются для фильтра в галерее — их не нужно регистрировать где-то ещё, фильтр строится автоматически из всех работ |
| `showcase` | Оставь `false` — для полной галереи (не для «Избранного» на главной) |
| `dateAdded` | Дата **публикации на сайте** в формате `YYYY-MM-DD` (например, `2026-07-05`) — **не дата создания картинки**, а дата, когда именно эта запись появляется на сайте. От неё считается бейдж «NEW» (показывается 14 дней с этой даты) |
| `w`, `h` | Пиксельные размеры **preview**-файла (не full!) — например, если preview вписался в 640×800, пиши `w: 640, h: 800`. Это нужно, чтобы карточка сразу занимала правильное место на странице и не «прыгала» при загрузке картинки |

**Как узнать точные `w`/`h`:** открой файл preview в любой программе
просмотра изображений (или в squoosh.app при экспорте — там показан
итоговый размер) и посмотри его реальные пиксельные размеры.

Не забудь запятую после `w: ..., h: ...,` — если это не последняя запись
в файле (а она обычно последняя, так что запятая не нужна перед `];`,
но нужна после предыдущей записи, если ты вставляешь не в конец).

## Шаг 4. Проверь локально и опубликуй

### 4.1 Проверка локально

1. Открой терминал в папке проекта, выполни:
   ```
   python -m http.server 8761
   ```
2. Открой в браузере `http://localhost:8761/gallery.html`
3. Убедись, что:
   - новая карточка появилась в галерее
   - картинка отображается (preview)
   - при клике на карточку открывается лайтбокс с **full**-версией
   - если у работы новые теги — они появились в фильтре сверху, и фильтр
     по ним работает
   - если работа опубликована в последние 14 дней — на карточке есть
     бейдж «NEW»

Если картинка не появилась — проверь в первую очередь:
- совпадает ли slug в файле `works.js` с именами файлов `.webp`
- нет ли опечатки в структуре (пропущенная запятая, скобка) — открой
  консоль браузера (F12 → Console), там будет ошибка JavaScript, если
  что-то не так с синтаксисом файла

### 4.2 Публикация на боевой сайт

Когда всё проверено локально, нужно отправить изменения на GitHub —
через минуту-две после этого обновится боевой сайт на GitHub Pages.

Если у тебя настроен git и есть доступ к репозиторию — из терминала в
папке проекта:

```
git add .
git commit -m "Add new gallery work: {slug}"
git push
```

Если это делаю я (ассистент) — просто попроси меня «запушить изменения»,
и я сделаю это за тебя.

## Если нужно добавить работу и в «Избранное» (showcase) на главной

Это отдельная задача посложнее — помимо всего вышеперечисленного,
потребуется:
- положить ещё и файл в `assets/works/showcase/{slug}.webp` (третий
  размер/версия для баннера на главной)
- поставить `showcase: true` вместо `false`
- добавить поле `accent: "gold"` или `accent: "cyan"` — акцентный цвет
  неоновой рамки карточки в блоке «Избранное» (должен сочетаться с
  остальными работами показа, лучше согласовать заранее, какой цвет
  использовать)
- убедиться, что на главной странице отображается не больше 4 showcase-работ
  (сейчас дизайн рассчитан на ровно 4 карточки в баннере и блоке «Избранное»)

Если понадобится — обратись за помощью отдельно, это лучше делать не
«на автомате» по инструкции, а с проверкой каждого шага.
